# 🧑‍🤝‍🧑 Workshop de Requisitos

## Quando usar
- Quando múltiplos stakeholders precisam **acordar juntos**
- Quando há divergência entre áreas
- Para acelerar levantamento (1 workshop = 5 entrevistas)

## Agenda-padrão (4 horas)

| Hora | Bloco | Técnica |
| :--- | :--- | :--- |
| 09:00 | Abertura + objetivo | Speech do sponsor |
| 09:15 | Alinhamento de contexto | SIPOC no quadro |
| 10:00 | Mapeamento As-Is | BPMN colaborativo |
| 10:45 | Coffee | — |
| 11:00 | Ideação To-Be | Brainstorming + agrupamento |
| 12:00 | Priorização | MoSCoW / dot voting |
| 12:45 | Próximos passos | Ata + responsáveis |

## Facilitação
- 1 facilitador (AF) + 1 escriba
- Post-its de cores diferentes por tema
- Estabelecer regras: "criticar ideia, não pessoa"
- Registrar TUDO em fotos/miro

## Saídas típicas
- ✅ Diagrama BPMN As-Is + To-Be
- ✅ Lista de requisitos priorizada (MoSCoW)
- ✅ Ata assinada
- ✅ Backlog inicial de user stories
