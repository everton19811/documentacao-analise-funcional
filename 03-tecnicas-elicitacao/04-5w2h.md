# ❓ 5W2H aplicado a Requisitos

Checklist para garantir que um requisito está **completo e não-ambíguo**.

| Sigla | Pergunta | Exemplo (RF-CON-002) |
| :--- | :--- | :--- |
| **What** | O que deve ser feito? | Conciliar automaticamente lançamentos |
| **Why** | Por que? Qual valor? | Reduzir 6h/dia de trabalho manual |
| **Where** | Onde acontece? | Módulo Conciliação, tela Nova Conciliação |
| **When** | Quando? Frequência? | Após upload; diariamente 23h (batch) |
| **Who** | Quem executa? Quem consome? | Analista Financeiro executa; Gestor consome relatório |
| **How** | Como funciona? Passo a passo | Sistema aplica RN-05 (valor + data ± 3d) |
| **How much** | Quanto custa? Qual volume? | ~100k lançamentos/mês; batch em < 60s |

## Uso prático
Antes de fechar qualquer US, o AF deve conseguir responder às 7 perguntas. Se falta 1, o requisito não está pronto para dev.
