---

## 📌 Descrição do Pull Request

Descreva de forma clara e sucinta as alterações incluídas neste PR. Explique qual funcionalidade foi implementada, ajuste realizado ou documento adicionado/atualizado.

---

## 🔗 Tarefa / Issue Relacionada

Associe este PR à issue correspondente no backlog (use palavras-chave do GitHub como `Closes #12` ou `Fixes #34` para fechamento automático):

* Refers to Issue: #

---

## 🛠️ Tipo de Mudança

Marque as opções que se aplicam a este PR:

- [ ] 📄 **Documentação** (Project Charter, EAP, Cronogramas, Relatórios)
- [ ] 🚀 **Nova Funcionalidade / User Story** (mudança que adiciona valor ao produto)
- [ ] 🐛 **Correção de Bug / Ajuste** (correção de algo que não funcionava conforme esperado)
- [ ] ⚙️ **Configuração / Governança** (templates, workflows, automações no repositório)
- [ ] 🧹 **Refatoração / Organização** (melhoria estrutural sem alterar comportamento)

---

## ✅ Checklist do Contribuidor / Autor

Antes de solicitar revisão, certifique-se de ter cumprido as etapas abaixo:

- [ ] As alterações foram testadas/revisadas localmente.
- [ ] Os **Critérios de Aceite** definidos na Issue associada foram completamente atendidos.
- [ ] A documentação relevante do projeto foi atualizada (ex: `README.md`, wikis, diagramas).
- [ ] Não há conflitos com a branch principal (`main` ou `develop`).

---

## 📸 Evidências / Anexos (opcional)

Adicione capturas de tela, diagramas atualizados, links de protótipos ou relatórios que comprovem a conclusão da entrega.

---

## 🔍 Checklist do Revisor (Project Manager / Tech Lead)

- [ ] Escopo entregue está alinhado com o planejado para a Sprint/Marco.
- [ ] Histórico de commits organizado e mensagens claras.
- [ ] Impactos em riscos ou cronograma foram atualizados, se aplicável.
