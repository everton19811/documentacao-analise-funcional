# 👤 2. Papel do Analista Funcional

## Responsabilidades principais

| # | Responsabilidade | Artefato relacionado |
| :--- | :--- | :--- |
| 1 | Elicitar necessidades de negócio | Atas de reunião, entrevistas |
| 2 | Traduzir necessidade em requisito funcional | BRD → FRD |
| 3 | Modelar processos As-Is e To-Be | BPMN, GAP Analysis |
| 4 | Escrever User Stories / Casos de Uso | US, UC |
| 5 | Documentar regras de negócio | Catálogo de Regras |
| 6 | Manter rastreabilidade requisito → teste | RTM |
| 7 | Apoiar QA na definição de casos de teste | Plano de UAT |
| 8 | Conduzir homologação com o usuário | Checklist UAT + sign-off |
| 9 | Gerir mudanças de escopo | Change Request |
| 10 | Manter o dicionário de dados / glossário | Glossário |

---

## Hard skills essenciais

- **Metodologias:** BABOK v3, PMBOK 7ed, Scrum, Kanban
- **Modelagem:** BPMN 2.0, UML (Casos de Uso, Atividade, Sequência), Fluxograma
- **Ferramentas:** Jira, Confluence, MS Project, Visio, Draw.io, Miro, Figma
- **Bancos de dados:** SQL básico/intermediário (para validar regras)
- **Documentação:** Markdown, Word, Excel avançado
- **Testes:** UAT, definição de cenários de teste (Gherkin - Given/When/Then)

## Soft skills essenciais

- 🗣️ Comunicação — traduzir "linguagem de negócio" ↔ "linguagem técnica"
- 👂 Escuta ativa — captar requisitos implícitos
- 🧩 Pensamento analítico — quebrar problemas complexos
- 🤝 Facilitação — conduzir workshops e resolver conflitos entre áreas
- 📋 Organização — gerenciar múltiplos requisitos simultâneos
